# Рабочий процесс: поддержание cursor_guide в актуальном состоянии

## Когда обновлять
- После выхода новых релизов Cursor (проверка `cursor.com/changelog`)
- При изменениях в структуре документации (`docs.cursor.com`)
- После появления новых возможностей в CLI / Agent / MCP

## Минимальный чек‑лист обновления
1. Обнови дату актуализации:
   - `README.md`
   - `cursor-project-setup-guide.md`
   - `FAQ.md`
   - `CHANGELOG.md`
2. Добавь/обнови разделы про новые функции (с официальными ссылками).
3. Проверь консистентность:
   - нет плейсхолдеров
   - все файлы/пути, упомянутые в тексте, реально существуют в репозитории
4. Обнови `cursor-config.json`:
   - `project.lastUpdated`
   - при необходимости — `project.version`
5. Добавь запись в `CHANGELOG.md` с датой и кратким списком изменений.
